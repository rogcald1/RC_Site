Projects (Updated 2/11/2021)

These are the projects I've worked on in order to get a good understanding of different programming languages. Do they look pretty? Probably not. But it was important to me that they worked as intended and that the user has a good experience (no errors or confusion). These can always use more work to look and function like a "real" site or app.

  • Rock Paper Scissor

    • Super early project with JavaScript and HTML. First usage of JS to adjust DOM elements. Code can most likely be written better to not be as lengthy, but still works overall.

  • To Do List Project

    • Mostly a JavaScript project, as well as a little HTML.

    I think for the most part, this one is fairly complete. Could always make more CSS design changes (and I may come back to it), but glad this works the way it should.

  • Calculator Project

    • Also a JavaScript project and HTML

    To me, infinitely trickier than the to-do project, and can definitely improve upon not only how the calculator works, but the JS code itself. One existing issue that I'll also have to figure out is the "NaN" response if the result length is too long. Also worth looking into adding the functionality of hitting the keys instead of clicking on each number.

  • Recipe App

    • Working with ReactJS this time, learning to call on an API. Tricky to put together but the hardest thing was finding the best API for this after I already started the project. Initial only had an "http://" version of their site, which caused security issues. Ended up going through RapidAPI for the info needed.

    Going to come back to this and add some formatting and design to it. Also need to add something to catch an issue with the API - seems to hit some errors occasionally.

  • Weather App

    • More ReactJS with API work here. Slightly easier than the recipe app only because you only need two things from the user instead of a list of ingredients. API was also easier to work with in comparison.

    I think this is also wrapped up for the time being. At least with what the weather API is able to provide. It'd be great if the app wasn't dependent on lat/long but zip code instead, so that would require finding another API to help convert zip to what is needed.
